2.1) Если мы постоянно используем поиск по месяцам, то можно разбить таблицы заказов на партиции по месяцам, чтобы запрос к БД сканировал только одну партицию, а не всю таблицу

2.2) Создать покрывающий индекс

2.3) Вычислять агрегацию заранее с помощью замены VIEW на MATERIALIZED VIEW, что позволит читать отчет за константное время относительно объема заказов

2.4) Добавить составной индекс, который позволит выполнить запрос не обращаясь к основной таблице:

```sql
CREATE INDEX idx_orders_month ON orders(created_at) INCLUDE (id);
```

CREATE INDEX idx_order_items_agg ON order_items(product_id, quantity);
